## 一、项目概述

### 1.1 背景

\"骗子酒馆\"是一款基于社交推理与欺骗的多人卡牌游戏，玩家通过虚张声势、真假混打出牌，在欢笑与互相质疑中决出最后的赢家。将这款线下桌游搬上微信小程序，旨在让朋友间随时随地开一局，降低线下凑局的门槛。

### 1.2 目标

- 开发一个支持 2-6 人实时对战的微信小程序
- 匹配/好友房两种开房方式
- 流畅的桌面动画与沉浸感 UI
- 一套清晰、自洽且易于上手的游戏规则

### 1.3 目标用户

- 18-35 岁，喜欢桌游/社交游戏的年轻人
- 朋友聚会、线上连麦场景
- 对规则长度敏感，希望 5 分钟上手、15 分钟一局

## 二、核心玩法规则

> 以下为 v1.0 采用的标准规则，后续版本可在此基础上扩展变体模式。

### 2.1 游戏准备

| 项目 | 说明 |
|------|------|
| 人数 | 2-6 人 |
| 牌组 | 标准 52 张扑克牌（去掉大小王） |
| 生命值 | 每位玩家 5 条命（用 ❤️ 显示） |
| 发牌 | 所有牌尽量均分发给每位玩家 |

### 2.2 回合流程

1. **目标点数** 按轮递增：A -> 2 -> 3 -> ... -> K -> A -> ...（循环）。每一轮所有玩家只围绕这一个点数出牌。

2. **起始玩家** 先出牌：
   - 选择手牌中 1-4 张牌，牌面朝下放在桌面中央
   - **必须声明** 这些牌就是当前目标点数（可以说真话，也可以说谎）

3. **下家（顺时针）三种行动选择：**

   | 行动 | 说明 |
   |------|------|
   | **质疑（开）** | 翻开上家打出的牌查验。若有任何一张与声明点数不符，出牌者扣 1 条命；若全部符合，质疑者扣 1 条命。查验后所有牌进入弃牌堆，本回合结束。 |
   | **跟牌** | 从手牌中再打 1-4 张牌面朝下放入中央堆，同样声称是当前目标点数。出牌权移交给下家。 |
   | **过** | 不做任何操作，出牌权移交给下家。 |

4. **一轮结束条件（触发其一则结束）：**
   - 有人被质疑且完成查验
   - 除当前出牌者外所有人都选择过 -> 当前出牌者本轮成功，所有牌堆进入弃牌堆

5. **一轮结束后：**
   - 目标点数 +1（A->2->3->...->K->A）
   - 扣命者从手牌中移除 1 张牌（如有），然后从扣命者的下家开始新一轮
   - 若无人扣命，从当前出牌者的下家开始新一轮

### 2.3 淘汰与胜利

- 玩家生命值归 0 时被淘汰，公开身份并退出当局
- 最后存活的一名玩家获胜，展示胜利结算界面

### 2.4 游戏流程状态机

\`\`\`
等待中 -> 发牌 -> 出牌(当前玩家) -> 下家选择(质疑/跟牌/过)
                                       |
                              (触发结束条件)
                                       |
                                  结算本轮
                                       |
                                  检查淘汰 -> 检查胜利
                                       |
                              (有胜者) -> 结算界面
                              (继续) -> 下一轮
\`\`\`

## 三、功能模块

### [P0] 3.1 房间系统

- [ ] **创建房间**：随机 6 位房间码，房主可设置玩家人数（2-6）
- [ ] **加入房间**：输入房间码加入；支持通过分享卡片直接跳转
- [ ] **房间内状态**：显示玩家列表（头像、昵称、准备状态），房主可开始游戏
- [ ] **准备/取消准备**：所有玩家准备后房主可开始
- [ ] **房间解散**：房主退出时自动解散；非房主退出回到大厅

### [P0] 3.2 游戏流程

- [ ] **发牌动画**：自动发牌，玩家看到自己的手牌
- [ ] **回合指示**：清晰标识当前轮到谁、当前目标点数、桌面中央牌堆数量
- [ ] **出牌操作**：选择 1-4 张手牌，确认打出并自动声明点数
- [ ] **下家行动面板**：展示三个按钮（质疑/跟牌/过），带倒计时（10-20s）
- [ ] **质疑动画与结果展示**：翻牌动画 -> 判定 -> 扣命/扣牌特效
- [ ] **淘汰播报**：淘汰时全屏提示，被淘汰玩家变观众
- [ ] **胜利结算**：展示胜者及当局数据统计

### [P1] 3.3 玩家信息

- [ ] **微信头像/昵称**：自动获取微信授权信息
- [ ] **局内状态展示**：剩余命数、手牌数量（对手仅显示数量）、玩家位置
- [ ] **简单战绩统计**：总局数、胜场

### [P2] 3.4 社交功能

- [ ] **微信分享**：分享房间卡片给好友/群聊
- [ ] **表情/快捷消息**：局内发送表情或预设文字
- [ ] **好友列表**：直接邀请好友

### [P1] 3.5 设置与辅助

- [ ] **操作倒计时**：默认 15s，房主可调节（10s/15s/20s/不限时）
- [ ] **音效开关**：背景音、出牌、质疑等音效
- [ ] **游戏规则页**：内嵌规则说明
- [ ] **举报与反馈**：违规举报入口

## 四、技术架构

### 4.1 整体架构

\`\`\`
+------------------+      WebSocket      +------------------+
|  微信小程序端     | <----------------> |  游戏服务端       |
|  (WeChat MP)     |     HTTP/WS        |  (Node.js + WS)  |
+------------------+                     +------------------+
\`\`\`

### 4.2 前端技术选型

| 技术 | 选择 |
|------|------|
| 框架 | 微信原生小程序（WXML + WXSS + JS） |
| 状态管理 | 全局 store（自定义简单状态管理） |
| 动画 | CSS animation + wx.createAnimation |
| 实时通信 | WebSocket（wx.connectSocket） |

### 4.3 后端技术选型

| 技术 | 选择 |
|------|------|
| 运行时 | Node.js |
| WebSocket | ws 库 或 Socket.IO |
| 数据持久化 | Redis（房间状态、在线玩家） |
| 部署 | 云服务器 / 微信云托管 |

### 4.4 数据流

\`\`\`
客户端操作 -> 发送 WS 消息 -> 服务端校验与更新状态
    -> 广播新状态给房间内所有客户端 -> 客户端渲染更新
\`\`\`

## 五、UI/UX 要求

### [P0] 5.1 页面结构

| 页面 | 说明 |
|------|------|
| 首页 | 游戏名称 + Logo + 创建房间/加入房间两个主按钮 |
| 房间页 | 玩家列表、准备按钮、房间码展示、分享按钮、开始按钮（房主） |
| 游戏页 | 座位布局、手牌、桌面、操作面板、信息区域 |
| 结算页 | 胜者展示、数据统计、再来一局/退出 |
| 规则页 | 图文规则说明 |

### [P0] 5.2 游戏页布局（关键）

\`\`\`
+--------------------------+
|    对手A [❤️x5] [牌x5]    |
|            |              |
|  对手B <-  桌  面  -> 对手D|
|            |              |
|    操作面板   对手C [...]  |
|                          |
|       [我的手牌区域]      |
+--------------------------+
\`\`\`

- 自己视角始终在底部，手牌扇形展开
- 对手显示头像、昵称、命数、手牌数量（不显示具体牌面）
- 桌面中央显示当前牌堆数量和当前目标点数
- 操作面板固定在底部操作区域

### [P1] 5.3 视觉风格

- **主题**：西部酒馆风格/深色暖调（棕色 + 金色 + 暗红）
- **牌面**：经典扑克背面，带\"骗子酒馆\"定制花纹
- **动画**：
  - 发牌从桌面中央飞到手牌区
  - 出牌从手牌飞到桌面中央并翻面
  - 质疑时翻牌 + 高亮 + 判定特效
  - 扣命时心碎动画 + 牌从手中移除
- **字体**：清晰可读的中文字体，重要的数字（点数、命数）突出显示

### [P2] 5.4 交互细节

- 选牌：点击选中/取消，最多选 4 张，选中后高亮
- 确认出牌：选牌后出现出牌按钮
- 质疑/跟牌/过：大按钮，带图标，颜色区分
- 倒计时：圆形进度条或数字倒计时，最后 5 秒变红闪烁

## 六、非功能需求

| 需求 | 说明 |
|------|------|
| 响应速度 | 操作到反馈 < 500ms |
| 断线重连 | 网络断开后 30s 内重连恢复游戏 |
| 弱网适配 | 支持 3G 网络下的基础流畅体验 |
| 防作弊 | 服务端校验所有出牌，客户端仅展示该看的信息 |
| 合规 | 接入微信登录、内容安全审核（用户昵称等） |
| 包体积 | 主包 < 2MB，使用分包加载 |
